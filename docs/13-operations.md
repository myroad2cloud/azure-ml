# 13. Operations

## Purpose

Operations helps the team understand whether the platform and deployments are healthy.

## What to track

- Pipeline status
- Job success and failure
- Compute usage
- Endpoint availability
- Endpoint latency
- Error rate
- Storage usage
- Cost trends

## Azure services

- Azure Monitor
- Log Analytics
- Application Insights
- Azure ML job history
- Diagnostic settings
- Cost Management

## Ownership questions

- Who owns failed training jobs?
- Who approves production model promotion?
- Who monitors endpoint health?
- Who handles incident response?
- Who manages cost optimization?

## Common mistakes

- Monitoring only the model and ignoring the platform
- No alert routing
- No cost review
- No endpoint ownership
- No runbook for failed deployments
