Learning path

The lessons taught so far.

1. Azure ML mental model

Azure ML is an enterprise ML platform, not just a notebook tool. It is the secure R&D workshop where experiments, jobs, compute, models, and endpoints are managed.

2. Secure target architecture

The confirmed pattern is Terraform-built, private-only Azure ML Workbench with model endpoint and controlled web UI access.

3. SOW delivery map

Delivery is accepted only when workspace, compute, experiments, model registry, endpoint, web UI, DevOps pipeline, and monitoring are validated.

4. Terraform and DevOps split

Terraform builds the factory. Azure ML CLI or SDK runs the factory.

5. Compute model

Compute Instance is the personal desk. CPU cluster is the shared machine. GPU cluster is the expensive training machine with min nodes set to zero.

6. MLflow, registry, endpoint

MLflow tracks experiments. Model Registry versions approved models. Managed Online Endpoint serves real-time predictions.

7. Azure Container Apps web UI

The web UI is a controlled access layer to test or consume the model endpoint. It is not a full enterprise app transformation.

8. Validation and handover

Close with evidence: jobs run, model registered, endpoint deployed, web UI working, monitoring visible, pipeline executed.

9. Workspace internals

Workspace is the control room. Storage, Key Vault, ACR, App Insights, Log Analytics, Private Endpoints, and Private DNS make it usable in the real world.

Workspace internals

Control plane vs data plane.

Control plane

  • Create workspace
  • Create compute
  • Submit jobs
  • Register models
  • Deploy endpoints
  • Assign permissions

Data plane

  • Read training data
  • Write outputs
  • Pull container images
  • Run training code
  • Serve predictions
  • Log telemetry

Architect warning: Workspace opening in Studio does not prove readiness. A full job must run, pull image, read/write storage, register model, deploy endpoint, serve prediction, and emit logs.

Handover

Definition of done.

Validation evidence

  • Workspace accessible from approved path
  • Compute Instance operational
  • CPU/GPU cluster functional
  • Experiment visible in MLflow
  • Model registered
  • Endpoint deployed
  • Web UI calls endpoint
  • Logs visible
  • Pipeline executed

Handover pack

  • Architecture design document
  • Configuration and deployment document
  • Operational runbook
  • Validation checklist
  • Known limitations
  • Cost controls
  • Knowledge transfer material

Client-safe line: The platform has been deployed and validated in the customer-provided Azure test environment. Ongoing operation, access policy ownership, security monitoring, and compliance reporting remain with the customer unless covered by a separate managed services agreement.
