1. Azure ML mental model

10. Compute sizing and cost

Use min nodes zero, constrained GPU scale, idle shutdown, quota validation, and tags.

Lesson 10

Compute sizing and cost controls.

Compute Instance

Personal development desk for notebooks, debugging, and small experiments. Control it with idle shutdown and modest sizing.

CPU Cluster

Default engine for feature engineering, classical training, batch scoring, and pipeline steps. Start with min 0 and max 2 or 4.

GPU Cluster

Use only for real GPU need. Start with min 0 and max 1. Validate quota and regional capacity early.

Cost rule

min_instances = 0 means no job, no running nodes. min_instances greater than 0 means nodes keep running even when idle.

ControlWhy it matters
Cluster minimum nodes set to 0Stops paying for idle training nodes
GPU maximum nodes kept lowPrevents accidental expensive scale-out
Idle shutdown for compute instancesPrevents cloud laptop cost leakage
Quota validation earlyAvoids delivery blockers during testing
Tags on all computeSupports cost reporting and ownership

Workspace internals

Control plane vs data plane.

Control plane

  • Create workspace
  • Create compute
  • Submit jobs
  • Register models
  • Deploy endpoints
  • Assign permissions

Data plane

  • Read training data
  • Write outputs
  • Pull container images
  • Run training code
  • Serve predictions
  • Log telemetry

Architect warning: Workspace opening in Studio does not prove readiness. A full job must run, pull image, write output, register model, deploy endpoint, serve prediction, and emit logs.

Handover

  • Workspace accessible from approved path
  • Compute Instance operational
  • CPU/GPU cluster functional
  • Experiment visible in MLflow
  • Model registered
  • Endpoint deployed
  • Web UI calls endpoint
  • Logs visible
  • Pipeline executed
